Get Started

Material CSSclose

Is a responsive front-end framework based on material design. It is provided in two versions, either compiled (materialize) or for pre-processing (sass).

Furthermore, you may find some community-made options (third-party). These may easily setup materialize in specific platforms/projects. Since, these are maintained by other parties, they are provided untested and may be out-of-sync with latest materializes versions.

PrimeFacesmore_vert

Documentation

PrimeFacesclose

PrimeFaces is an open source component library for JavaServer Faces (JSF) based applications. PrimeFaces is created by PrimeTek. Besides community editions and themes, you may also buy extended themes.

Besides JSF, PrimeFaces also offers libraries to extend JavaScript-based applications. The latter is provided in three versions, named:

  1. PimeNG (for Angular)
  2. PimeReact (for React)
  3. PimeUI (for JavaScript in general)

JSF - Java Server Facesmore_vert

Specification

Card Titleclose

JavaServer Faces is a framework technology intended to simplify building of user interfaces for Java EE applications. Developers can build web applications by assembling reuseable UI components in a page; connecting these components to an application data source; and wiring client-generated events to server-side event handlers.

There are two implementations of the JSF specification, both of them developed with OSI approved Open Source licenses.

  • Mojarra
  • Apache MyFaces

During configuration of a JSF application, you should keep an eye on schema definitions: JSF Schemas. Although NetBeans generates most of them, they may not address latest versions all the time. Currently JSF is standardized in version 2.3 but not yet supported in NetBeans generators (maybe because Java EE 8 is not finalized yet).

Grid Layout Explained

Responsive Designclose

Responsive web design (RWD for short) is a design and technical paradigm for the creation of websites, so that they can react to properties of the end device used, especially smartphones and tablet computers. The graphical structure of a responsive website is based on the requirements of the respective device with which the website is viewed. This applies in particular to the arrangement and display of individual elements, such as navigation, page columns and texts, but also to the use of different input methods of mouse (click, run over) or touch screen (type, wipe). The technical basis for this are web standards such as HTML5, CSS3 (here especially the media queries) and JavaScript.

In order to create a responsive design, a grid like structure that is controlled by CSS is usually used. This is often also called Grid CSS. Such a layout can be created relatively easily using JSF or Materialize tools:
